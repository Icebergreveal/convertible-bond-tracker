# AI Coding Agent 使用规则

## 角色定义

AI coding agent 是开发助理，不是项目负责人。

### 学生职责
- 定义金融问题
- 判断公告数据是否匹配
- 设计字段、schema、验收标准
- 运行代码、检查输出、记录错误
- 解释项目方法与结果

### AI 职责
- 生成项目骨架
- 写小模块代码
- 解释报错
- 补充日志与测试
- 重构重复代码
- 帮助整理 README 与展示材料

## Vibe Coding 流程

```
Spec -> Plan -> Generate -> Run -> Repair -> Review
```

1. **Spec**：明确规格定义
2. **Plan**：制定实现计划
3. **Generate**：AI生成代码
4. **Run**：本地运行测试
5. **Repair**：修复报错和问题
6. **Review**：检查结果并记录

## 编码规范

### 命名规则
- 文件名：snake_case.py
- 变量名：snake_case
- 函数名：snake_case
- 类名：PascalCase
- 常量：UPPER_SNAKE_CASE

### 代码结构
- 每个模块职责单一
- 函数不超过50行
- 类不超过200行
- 合理使用类型提示

### 日志要求
- 关键步骤必须记录日志
- 错误必须记录详细信息
- 日志格式统一

### 安全规范
- API Key 必须放入 .env
- 敏感信息不得硬编码
- 不绕过访问限制
- 抓取必须限速

## AI 使用记录要求

每次使用AI后必须在 `ai_worklog_all.md` 中记录：
- 时间戳
- 请求内容
- AI响应摘要
- 运行结果
- 人工验证情况
- 错误与修复记录

## 项目特定规则

### 数据规则
- 主数据必须来自巨潮资讯网公开公告
- 不绕过登录、验证码或访问限制
- 原始PDF不得修改

### 抽取规则
- 字段值必须来自公告文本
- 每个关键字段要有 evidence_text
- 无法判断时输出 null
- 抽取结果必须通过 Pydantic 校验

### 可转债特定字段
- 转股价格单位统一为元/股
- 日期格式统一为 YYYY-MM-DD
- 金额单位统一处理（元/万元/亿元）
- 下修幅度 = (修正前转股价 - 修正后转股价) / 修正前转股价 * 100%
- 赎回溢价率 = (赎回价格 - 转股价值) / 转股价值 * 100%
