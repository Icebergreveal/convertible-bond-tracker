# AI Usage Statement

## 使用的 AI coding agent

本项目使用 TRAE AI coding agent 作为开发助理。

## AI 帮助完成的内容

1. **项目骨架设计**：帮助设计项目目录结构、模块划分和文件组织
2. **代码生成**：生成爬虫模块、PDF解析模块、Section定位模块、LLM抽取模块、事件匹配模块等
3. **配置文件**：生成各类配置文件（YAML格式）
4. **课程文档**：生成 topic_proposal.md、crawl_spec.md、difficulty_declaration.md、workflow_design.md、eval_report_template.md 等
5. **代码解释**：解释代码逻辑和运行方式

## 学生人工完成的内容

1. **金融问题定义**：定义可转债下修与强赎双事件分析的金融问题
2. **Schema设计**：设计17个结构化字段及其金融含义
3. **数据来源确认**：确认巨潮资讯网数据来源和公告类型
4. **难度档位申请**：分析任务复杂度并申请挑战档1.1
5. **人工评估**：对抽取结果进行人工抽样检查
6. **结果解释**：解释项目方法与结果

## 我们如何验证 AI 生成内容

1. **代码审查**：逐模块审查AI生成的代码，确保逻辑正确
2. **运行测试**：运行单步测试命令验证各模块功能
3. **结果校验**：使用Pydantic强制校验输出格式
4. **人工复核**：对关键结果进行人工抽样检查
5. **日志追踪**：记录所有运行日志便于问题定位

## 发现过的 AI 错误

1. **字段命名不一致**：需要统一字段命名规范
2. **日期格式不统一**：需要添加日期标准化处理
3. **数值精度问题**：需要统一价格保留2位小数
4. **边界条件处理**：需要补充空值处理逻辑

## API Key 与数据合规说明

1. API Key 存储在 .env 文件中，不提交到版本控制
2. 只抓取巨潮资讯网公开可访问的数据
3. 遵守robots.txt协议，设置合理的请求间隔
4. 保留完整的抓取和处理日志
