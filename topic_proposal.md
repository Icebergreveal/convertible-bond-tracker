# Topic Proposal

## 项目题目

可转债转股价下修与提前强赎双事件全生命周期跟踪及结构化分析

## 金融问题

可转债是上市公司重要的融资工具，转股价下修、提前强赎是存续期内最核心的两大资本运作事件。目前巨潮资讯网相关公告分散、文本非结构化，人工整理耗时费力，难以快速梳理事件时间线、提取关键条款与价格数据，也无法批量量化下修幅度、赎回溢价等核心指标。

本项目旨在通过金融文本智能技术，自动归集可转债下修、强赎多类公告，完成PDF解析、章节定位、关键信息结构化抽取、数值标准化与跨公告事件匹配，构建完整事件时间线，量化核心金融指标，实现可转债重大事项的自动归集、可追溯、可对比的结构化分析。

## 巨潮数据来源

全部数据仅来源于巨潮资讯网（cninfo.com.cn）A股上市公司公开发行可转债的官方信息披露PDF公告。

## 公告类型

### 一、转股价下修相关公告
1. 可转债触发转股价向下修正条件提示公告
2. 董事会关于转股价向下修正的提议公告
3. 股东大会审议转股价下修决议公告
4. 转股价向下修正实施公告

### 二、可转债提前强赎相关公告
1. 可转债满足提前赎回条件提示公告
2. 公司行使可转债提前赎回权决议公告
3. 可转债提前赎回实施提示公告
4. 可转债提前赎回结果暨摘牌公告

| stock_code | 字符串 | 是 | 正股代码 | 公告标题 |
| stock_name | 字符串 | 是 | 正股名称 | 公告标题 |
| bond_code | 字符串 | 是 | 转债代码 | 公告标题 |
| bond_name | 字符串 | 是 | 转债简称 | 公告标题 |

| publish_date | 日期 | 是 | 公告日期 | 落款日期 |

| evidence_text | 字符串 | 是 | 证据原文 | MinerU解析原文 |

## 最终输出形式

1. metadata.csv - 公告信息台账
2. 解析结果 - Markdown文本、章节定位报告
3. 结构化数据 - JSON/CSV格式抽取结果
4. 事件时间线 - 50+组完整事件链
5. 量化指标 - 下修幅度、赎回溢价率统计表
6. 工作流 - 一键运行入口、日志、评估报告

## 难度档位
- 系数：1.1
- 理由：150+ PDF公告，50+事件链，8种公告类型，多文档匹配，17个字段，符合挑战档标准

## 风险与备选方案
1. PDF解析错乱 → 人工复核校正
2. LLM幻觉 → Pydantic校验 + null规则
3. 多事件混淆 → 转债代码+时间窗口匹配
4. 标题不规范 → 关键词双重过滤
